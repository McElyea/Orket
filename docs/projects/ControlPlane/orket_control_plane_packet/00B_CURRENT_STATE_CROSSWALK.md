Last updated: 2026-03-24
Status: Accepted for implementation planning

## Purpose

Map current Orket runtime surfaces to the control-plane nouns in this packet.

This document exists to prevent clean-room architecture drift.
It is an honest migration map, not a claim that the current runtime already implements the full control plane.

## Current-state conclusion

The strongest current alignment exists around:
1. run identity and run-start artifacts
2. sandbox execution, resource, reservation, reconciliation, lease, operator-command, and final-truth publication services on the default orchestrator path
3. sandbox operator views now expose control-plane run state, attempt state, reservation and lease status, final-truth summary, latest operator command summary, and the latest reconciliation id, divergence class, and safe-continuation class when durable reconciliation truth exists
4. repo-backed tool-approval requests, guard-review pending gates, and governed kernel approvals now publish explicit operator-hold reservation truth on request creation or admission and resolve that reservation on approval decision, with approval list and detail surfaces exposing the latest reservation summary on both the approval surface and the named governed target when present, plus the latest target-side run and attempt summary when durable execution truth exists including policy/config snapshot ids, creation timestamp, and attempt count, the latest target-side step, checkpoint, and effect-journal summary when durable target truth exists including step capability, checkpoint acceptance supervisor or dependency detail, and effect publication lineage or digest detail, richer target-side operator summaries including receipt refs and affected transition/resource refs, the latest target-side final-truth summary when durable closure truth exists, and governed kernel approvals can now recover that target from reservation truth instead of relying on a copied payload field
5. authenticated tool-approval resolutions now publish explicit operator records on the default API-to-engine approval path, with grants recorded as risk acceptance and denials or expirations recorded as terminal commands
6. authenticated kernel session-end now publishes explicit operator cancel commands for governed kernel-action runs on the default API path
7. the governed kernel action API path now publishes per-trace run, attempt, commit-step, effect-journal, and final-truth authority for admit, commit, and session-end closure
8. kernel replay and audit summaries now surface latest step identity plus namespace scope, resources touched, and receipt refs, latest reservation summary including invalidation basis and supervisor authority, and closure state alongside run, attempt, effect, final-truth, and operator-action summaries, while approval-derived views on governed kernel targets can also expose policy/config snapshot ids, creation timestamp, attempt count, and effect publication detail through the same durable records
9. the sandbox reclaimable path now publishes checkpoint-backed new-attempt recovery decisions that target the accepted reclaimable checkpoint directly, with live operator views preserving the latest recovery decision and checkpoint summary across attempt rollover after reacquire

The weakest areas, and therefore the primary implementation risks, are:
1. reservation truth now covers sandbox allocation, approval-gated operator holds for repo-backed tool approvals, guard-review pending gates, and governed kernel approvals, standalone coordinator claims, and Gitea worker claim admission, but it is still not wired into general admission and scheduling
2. lease truth is no longer sandbox-only, and the standalone coordinator API now publishes non-hedged lease history, but it is still not shared by admission, scheduling, or most non-sandbox runtime paths
3. operator-action authority is still fragmented in live runtime behavior outside explicit authenticated sandbox stop, approval-resolution publication, and authenticated kernel session-end cancel
4. effect truth is still often reconstructed from artifacts outside the sandbox, governed kernel action, and governed turn-tool paths instead of being published through the normative effect journal
5. checkpoint publication is now live on the sandbox reclaimable path, the governed turn execution pre-effect path, and the Gitea worker pre-effect path, checkpoint-backed new-attempt recovery is now live on the sandbox reclaimable path and on unfinished pre-effect governed turn `resume_mode`, but same-attempt checkpoint resume and broader supervisor-owned checkpoint creation still are not execution-default behavior
6. namespace and safe-tooling gates are now live on the governed turn-tool path, but broader runtime workloads, scheduling, and non-governed tooling still do not share one explicit namespace authority surface
